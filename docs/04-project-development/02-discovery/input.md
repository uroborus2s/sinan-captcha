 # 项目输入

- 项目名称：sinan-captcha
- 中文名：司南
- 当前阶段：REQUIREMENTS
- 核心创意：训练两个专项模型识别行为验证码，第一专项为图形点选多类别检测模型，第二专项为滑块缺口定位模型
- 用户条件：已有一台 Windows 电脑和 NVIDIA 显卡；有一定 IT 基础；没有模型训练经验
- 本轮目标：给零基础用户落实一套可执行的训练方案，重点覆盖训练环境搭建、样本获取、自动标注、训练、验收与后续迭代

## 已知约束

- 当前目录非空，不能按空仓方式直接 `factory-init`
- 当前仓库只有一份问题定义文档，没有现成代码和数据集
- 本轮工作停留在需求阶段，不进入实现阶段
- 用户明确要求“自动标注优先，而不是人工逐张标注”

## 2026-04-04 追加输入

- 在现有 `sinan-generator` 与 `sinan` CLI 基础上，补一条“AI 自主训练”主线
- 自主训练第一版必须先支持接入 `opencode`
- 结果查看、结果压缩、评估判断、下一轮建议应拆成 skill 或等价职责单元，避免长训练导致上下文爆炸
- 第一组与第二组都要纳入自主训练体系，但各自保持独立 study、独立指标和独立停止规则
- 自主训练仍要保持“确定性执行归 Python 控制器、解释与决策归 AI agent”的边界，不能退化为自由发挥式 shell 代理

## 外部参考的吸收结论

- `dddd_trainer` 适合借鉴 Windows/NVIDIA GPU 训练、自建训练项目和新手安装顺序，但它的主任务仍是 OCR 训练。
- `captcha_trainer` 适合借鉴零基础、Windows GPU 和项目化管理思路，但其模型场景以图像分类为主，不应直接作为当前图形定位任务的模型路线。
- 当前项目的正式训练主线应以目标检测为中心，而不是字符分类或 OCR 解码。

## 本轮最重要的问题

- 如何让零基础用户搭好 Windows + NVIDIA 训练环境
- 如何从自有验证码生成链路导出高质量标签
- 如何把图形点选和滑块两类样本统一纳入一个受控生成器
- 如何保证生成器输出的训练样本 100% 来源于内部真值，而不是后验猜测
- 如何定义两个专项模型的验收门槛
- 如何让整个流程可以被复现、回滚和继续迭代
- 如何在不依赖长对话上下文的前提下，让 AI 自动完成“生成样本 -> 训练 -> 测试 -> 判断 -> 继续/放弃”的闭环
