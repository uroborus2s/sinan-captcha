# 开发者指南概览

本目录回答的是：

- 维护者如何接手这个双 CLI 工程
- 如何理解仓库结构和运行目录边界
- 如何在本地修改、验证、打包和交付

## 1. 这部分和用户指南有什么区别

用户指南只回答：

- 怎样在 Windows 训练机上安装、生成数据、训练和验收

开发者指南回答的是：

- 怎样维护这套工程本身

所以这里会重点讲：

- 仓库结构
- Git 边界
- 本地开发工作流
- 发布与交付流程

而不会重复讲用户怎样训练。

## 3. 当前稳定事实

当前项目已经收口为两个正式 CLI：

- `sinan-generator`
  - Go CLI
  - 负责工作区、素材和数据集生成
- `sinan`
  - Python CLI
  - 负责训练目录、训练、评估、发布和交付

当前最重要的目录边界有 3 个：

- 源码仓库
- 生成器工作区
- 训练目录

当前最重要的交接边界有 2 个专项合同：

- `group1`：YOLO 数据集目录
- `group2`：paired dataset 目录

## 4. 这部分文档的使用原则

- 先看本目录，再决定是否需要跳到内部设计文档
- 先按当前实现和当前 CLI 写文档，不补历史故事
- 任何对外行为变更，都要回写用户指南
- 任何维护流程变更，都要回写开发者指南和 `.factory`
